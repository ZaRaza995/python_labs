# Лабораторная работа №9. "База данных" на CSV

## Описание
Реализован простейший менеджер данных студентов с использованием CSV-файла в качестве хранилища.
Реализованы основные CRUD-операции (Create, Read, Update, Delete), а также сбор статистики.

## Структура проекта

- `src/lab09/group.py`: Основной класс `Group`. Содержит логику работы с файлом и данные студентов.
- `src/data/lab09/students.csv`: Файл-хранилище данных.

## Основные возможности класса Group
- **Инициализация**: Автоматическое создание файла хранилища, если он отсутствует.
- **Добавление**: Метод `add(student)` добавляет новую запись.
- **Просмотр**: Метод `list()` возвращает список всех студентов.
- **Поиск**: Метод `find(substr)` ищет студентов по подстроке в ФИО.
- **Обновление**: Метод `update(fio, **fields)` позволяет изменить поля существующего студента.
- **Удаление**: Метод `remove(fio)` удаляет запись по ФИО.
- **Статистика**: Метод `stats()` возвращает словарь с агрегированными данными (количество, средний балл, распределение по группам и т.д.).

## Запуск
Для запуска интерактивного CLI (меню управления):

```bash
# Из корня проекта
python -m src.lab09.group
```

При запуске будет доступно текстовое меню:

1. Список студентов
2.  Добавить студента
3.  Найти
4.  Удалить
5.  Обновить
6.  Статистика
0.  Выход

## Пример использования в коде

```python
from src.lab09.group import Group
from src.lab08.models import Student

# Создание экземпляра группы
group = Group("src/data/lab09/students.csv")

# Добавление студента
new_student = Student("Иванов Иван", "2000-01-01", "101", 4.5)
group.add(new_student)

# Получение статистики
print(group.stats())
```

## Результаты
Реализован полноценный класс, который умеет работать с CSV файлом как с базой данных, обеспечивая корректность и сохранность информации между запусками программы.
